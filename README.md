# DigiCanji v1.2.0

手書き認識を利用した漢字学習アプリと、配布用クイズHTMLを生成するエディターです。

## ファイル

- `index.html` — DigiCanjiプレイヤー。GitHub Pagesのトップページとして公開できます。
- `editor.html` — 最大1000問を編集し、問題データ同梱の単一HTMLを生成するエディターです。
- `THIRD_PARTY_NOTICES.md` — 同梱データの第三者ライセンスに関する通知です。

## 公開

4ファイルをリポジトリの公開対象ルートへ置き、GitHub Pagesを有効にします。ビルド処理は不要です。

エディターで生成したクイズを公開する場合は、生成HTMLを任意のリポジトリへ置き、トップページにする場合は `index.html` へ改名してください。

## 問題データ

```text
前部,問部,後部,解答,解説
```

- 最大1000問
- 1回の出題は最大10問、重複なし
- 正答率が低い問題と未出題問題を優先する重み付きランダム
- 解答がひらがなのみの場合はフリー認識枠
- 問部と解答が同一、または解答にひらがな・漢字以外を含む場合はエラー

## v1.2.0の認識基盤

### 複数認識器の統合

ネイティブ認識器とGoogle系認識の候補を、単純な先着順ではなく同じ評価表へ統合します。両方の認識器に現れた候補を優先しつつ、画数、筆跡配置、構造検査も併用します。ネイティブ候補が一応成立していても確信度が低い場合は、Google候補と比較します。

### 筆跡の正規化再認識

通常座標の候補が得られない場合、筆跡の縦横比を保ったまま中央へ拡大・整列した座標でも再認識します。小さく書いた字、端へ寄った字、端末ごとのパッド寸法差に対応するための一般処理であり、特定の文字には依存しません。

### 文字種の拡張

- カタカナとして返されたひらがな筆跡を、フリーモードではひらがな候補へ正規化します。
- Unicodeの漢字判定をBMP内だけでなくCJK拡張領域まで広げました。
- 内蔵画数辞書にないまれな漢字も、認識器の上位候補または複数認識器の一致があれば候補として扱います。

### なぞり書き認識

なぞり書きでも、ネイティブ候補が誤っているだけで判定を終了せず、Google候補と正規化座標の候補まで確認します。模範解答と一致する候補がどこかに含まれる場合は、その候補を優先します。

### 字体

問題文、文字枠・フリー認識欄、なぞり書きは、共通の教科書体優先スタックを使います。文字ごとの独自ベクター字体は使用しません。

## 認識診断モード

公開URLの末尾に `?recognitionDebug=1` を付けると、設定画面に「認識診断」が表示されます。

端末内に最大80件まで保存します。

- 実筆座標
- ネイティブ、Google通常座標、Google正規化座標の候補列
- 統合候補の得点、認識器間の一致、拒否理由
- 実画数・仮想画数・許容画数範囲
- 構造検査の結果
- 教科書体候補の検出結果
- 端末・画面・利き手などの実行条件

保存したJSONまたは別端末で収集したJSONを読み込み、外部認識サービスへ再送せず現在の判定ロジックで一括再評価できます。再評価結果はJSONとして出力され、正答数、候補なし、正答候補の誤拒否などを確認できます。

通常URLでは診断記録を保存しません。

## 動作上の注意

認識器が正しい候補を一つも返さない場合、アプリ側の後処理だけでは完全には復元できません。v1.2.0では候補源と入力座標の多様化によってこの確率を下げていますが、端末別・書き手別の診断JSONによる継続的な校正は必要です。
