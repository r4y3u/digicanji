# DigiCanji v0.3.1

手書き漢字認識を使った学習用クイズアプリと、問題データを編集して配布用HTMLを生成するエディターです。

## 収録ファイル

- `DigiCanji.html`
  - テスト問題を同梱した単一HTML版プレイヤー
  - CSS・JavaScript・問題データを1ファイルに内包
- `DigiCanji_Editor.html`
  - DigiCanji専用エディター
  - 問題編集、CSV入出力、編集用プロジェクト保存、単一HTML生成に対応
- `source/`
  - プレイヤーの分割ソース
- `THIRD_PARTY_NOTICES.md`
  - 第三者由来データ等の表記

## v0.3.1の変更点

### エディター v0.2.0

- 編集可能な問題数を100問から1000問へ拡張
- 問部と解答が同じ場合にエラー表示
- 解答に、ひらがな・漢字以外が含まれる場合にエラー表示
- エラーのあるセルを強調し、セルのツールチップとプレビューに理由を表示
- 1000問を超えるCSV・プロジェクトは、先頭1000問を読み込み、省略を通知
- ひらがなのみの解答を、プレビュー上で「フリー」認識として表示

### プレイヤー v0.3.0

- 解答がひらがなのみの場合、文字数ごとの認識枠ではなく1つの「フリー」認識枠を使用
- フリー認識では複数文字のひらがな候補を保持
- 認識枠の位置表示を `フリー` に変更
- 縦書きでもフリー認識結果を縦方向に表示

## 入力検査

各問題について、次の条件を確認します。

- 問部が入力されている
- 解答が入力されている
- 問部と解答が同じではない
- 解答が、ひらがなと漢字だけで構成されている

解答がひらがなのみの場合は、読みを問う問題として扱い、プレイヤーの認識枠を `フリー` にします。

```csv
前部,問部,後部,解答,解説
改善を,ハカル,。,図る,しようと考えること
,アツイ,本を読む。,厚い,物を横から見たときの幅がある
「厚い」の読みを,答えなさい,。,あつい,「厚い」は「あつい」と読む。
```

解答欄には句読点、空白、数字、記号、カタカナを入力できません。句読点は後部へ入力してください。

## 基本ワークフロー

1. `DigiCanji_Editor.html` をブラウザで開く
2. 表へ問題を入力、またはCSVを読み込む
3. エラー表示がないことを確認する
4. タイトルとクイズIDを設定する
5. `HTML生成` を押す
6. 生成されたHTMLを配布またはGitHub Pagesへ配置する

プレイヤーにはCSV読込機能を設けていません。問題データは生成時にHTMLへ同梱されます。

## キーボード操作（プレイヤー）

### 横書き

- `← / →`：文字枠移動
- `↑`：見直し
- `↓`：進む／回答

### 縦書き

- `↑ / ↓`：文字枠移動
- 右利き：`←` が進む／回答、`→` が見直し
- 左利き：`→` が進む／回答、`←` が見直し

### 共通

- `1～9 / 0`：1～10問目へ移動
- `Z`：1画消す
- `Esc`：メニューやオーバーレイを閉じる

フリー認識問題では文字枠移動は行いません。

## 確認項目

- JavaScript構文検査
- エディターに1000行が表示されること
- 問部と解答が同一の場合のエラー表示
- 解答に句読点が含まれる場合のエラー表示
- ひらがな解答のフリープレビュー
- ひらがな解答を同梱したプレイヤーで、認識モードが `free` になること
- プレイヤーとエディターで実行時エラーが発生しないこと
