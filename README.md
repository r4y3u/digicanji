# DigiCanji v1.1.0

手書き認識を利用した漢字学習アプリと、配布用クイズHTMLを生成するエディターです。

## ファイル

- `index.html` — DigiCanjiプレイヤー。GitHub Pagesのトップページとしてそのまま公開できます。
- `editor.html` — 最大1000問を編集し、問題データ同梱の単一HTMLを生成するエディターです。
- `THIRD_PARTY_NOTICES.md` — 同梱データの第三者ライセンスに関する通知です。
- `README.md` — 本ファイルです。

## 公開

この4ファイルをリポジトリの公開対象ルートへ置き、GitHub Pagesを有効にします。ビルド処理は不要です。

エディターで生成したクイズを公開する場合は、生成HTMLを任意のリポジトリへ置き、トップページにする場合は `index.html` へ改名してください。

## 問題データ

エディターの列順は次のとおりです。

```text
前部,問部,後部,解答,解説
```

- 最大1000問
- 1回の出題は最大10問、重複なし
- 正答率が低い問題と未出題問題を優先する重み付きランダム
- 解答がひらがなのみの場合はフリー認識枠
- 問部と解答が同一、または解答にひらがな・漢字以外を含む場合はエラー

## v1.1.0の認識基盤

- ひらがなの画数判定を、文字ごとの例外表から語全体の一般式へ変更しました。活字の基準画数と学校で用いる手書き字形の差を、語長と基準画数から算出します。
- 濁点・半濁点は短縮可能な画数に含めず、実際の追加筆画を必須にします。
- 候補を即座に除外するだけでなく、画数差、候補順位、文字配置の支持度を得点化して比較します。
- ネイティブ認識器から取得する候補数を増やし、最上位候補が検査に落ちた場合も後続候補を評価します。
- 構造検査を文字ごとの条件分岐から部品規則の登録方式へ変更しました。`失`、`重`、`攵`など同じ部品を持つ複数文字へ同じ検査を適用できます。
- 問題文、認識欄、なぞり書きは、端末標準の教科書体を共通の優先順で使用します。字ごとの独自ベクター補正は廃止しました。

## 認識診断モード

公開URLの末尾に `?recognitionDebug=1` を付けると、設定画面に「認識診断」が表示されます。

診断モードでは、端末内に最大40件まで次の情報を保存し、JSONとして書き出せます。

- 実筆座標
- 生の候補列
- 実画数・仮想画数・許容画数範囲
- 候補ごとの得点と拒否理由
- 構造検査の結果
- 端末・画面・利き手などの実行条件

通常URLでは診断記録を保存しません。

## 動作上の注意

手書き認識は、対応ブラウザのHandwriting Recognition APIを優先し、利用できない場合はネットワーク経由の認識へフォールバックします。ネットワーク状態や外部サービスの仕様変更によって認識できない場合があります。

認識の一般化によって個別例外は減らしましたが、外部認識器が正しい候補を一つも返さない場合は、アプリ側だけでは復元できません。診断JSONを利用して、端末別・書き手別の実筆コーパスを増やしながら閾値を校正してください。
