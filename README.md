# DigiCanji v1.5.0

手書き認識を利用した漢字学習アプリと、配布用クイズHTMLを生成するエディターです。

## GitHubリポジトリへ置くファイル

- `index.html` — `kanji_sample.html`を開くための軽量な入口です。
- `kanji_sample.html` — プロジェクト「4年　全」から生成した800問のクイズアプリです。
- `editor.html` — 最大1000問を編集し、問題データ同梱の単一HTMLを生成するエディターです。
- `README.md` — 配置方法と更新内容です。
- `THIRD_PARTY_NOTICES.md` — 同梱データの第三者ライセンスに関する通知です。

5ファイルをリポジトリの公開対象ルートへ置き、GitHub Pagesを有効にします。ビルド処理は不要です。

## v1.5.0の回答方式

### 全問題を1枠1文字へ統一

書き問題だけでなく、読み問題も1枠1文字で認識します。通常の問題では文字列全体を一度に認識しないため、縦書きで起きていた文字順の入れ替わり、文字数の増減、濁点の所属先の誤判定を回避できます。

### 常に10枠を表示

正答の文字数に関係なく、すべての問題で10枠を用意します。正答長から送り仮名を推測できません。未入力の末尾枠は採点時に無視し、余分な文字を入力した場合は不正解になります。

### 正方形のスクロール式認識欄

- 横書きレイアウト：正方形枠を横1列に並べ、横スクロール
- 縦書きレイアウト：正方形枠を縦1列に並べ、縦スクロール
- 枠のクリック・タップ、矢印キーで選択
- 選択した枠は自動的に表示範囲へ移動

## 認識処理

- 読み問題では、カタカナ候補をひらがなへ正規化します。
- 読み問題では、1文字のひらがな候補だけを採用します。
- 書き問題では、漢字・かなを含む1文字候補を従来どおり画数・構造・認識器間一致で評価します。
- 通常認識が確実な場合は外部再認識を行わず、通信と処理を抑えます。

## 問題データ

```text
前部,問部,後部,解答,解説
```

- 最大1000問
- 1回の出題は最大10問、重複なし
- 正答率が低い問題と未出題問題を優先する重み付きランダム
- 問部と解答が同一、または解答にひらがな・漢字以外を含む場合はエラー

## 認識診断モード

公開URLの末尾に `?recognitionDebug=1` を付けると、設定画面に「認識診断」が表示されます。

端末内に最大80件まで、筆跡、各認識器の候補、得点、拒否理由、画数、使用可能フォントなどを保存できます。診断JSONは自動送信されません。
