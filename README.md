# DigiCanji v1.4.0

手書き認識を利用した漢字学習アプリと、配布用クイズHTMLを生成するエディターです。

## GitHubリポジトリへ置くファイル

- `index.html` — `kanji_sample.html`を開くための軽量な入口です。
- `kanji_sample.html` — 添付プロジェクト「4年　全」から生成した800問のクイズアプリです。
- `editor.html` — 最大1000問を編集し、問題データ同梱の単一HTMLを生成するエディターです。
- `THIRD_PARTY_NOTICES.md` — 同梱データの第三者ライセンスに関する通知です。

4ファイルをリポジトリの公開対象ルートへ置き、GitHub Pagesを有効にします。ビルド処理は不要です。

## v1.4.0の認識改善

### 縦書きフリー入力の濁点・半濁点判定

従来は濁点位置の検査が横書き前提に近く、縦書きで「かくじ」のように正しく書いても候補を落とす場合がありました。現在は筆跡全体の向きを判定し、次の位置関係を別々に評価します。

- 横書き：各文字の右上
- 縦書き：各文字の上部かつ右側

濁点は、単に短い線が2本あるだけでなく、互いに近く、方向と長さが大きく外れていないことも確認します。これにより、文字本体の短い画を濁点として数える誤判定を抑えます。

### 文字領域の自動分割

文字列を機械的な等間隔に分けず、筆跡のまとまりから文字領域を推定します。文字間隔が不ぞろいな縦書き・横書きでも、画数、濁点、小書き文字の証拠を対応する文字へ割り当てやすくしました。

つなげ書きで筆跡のまとまりが少ない場合は即時拒否せず、極端に文字数と筆跡量が合わない場合だけ除外します。

### 楷書・教科書体を優先した表示

問題文、認識欄、なぞり書きは同じフォント方針へ統一しました。

1. UDデジタル教科書体の楷書系
2. BIZ UDPゴシック
3. OS標準の日本語角ゴシック

明朝体と游教科書体をフォールバックから外し、うろこや環境依存のつなぎ字を避けています。Webフォントは通信・容量・ライセンス上の負担を増やすため同梱せず、端末内フォントだけを使用します。なぞり書きの模範字は合成太字を使わず、通常の字形で描画します。

## 問題データ

```text
前部,問部,後部,解答,解説
```

- 最大1000問
- 1回の出題は最大10問、重複なし
- 正答率が低い問題と未出題問題を優先する重み付きランダム
- 解答がひらがなのみの場合はフリー認識枠
- 問部と解答が同一、または解答にひらがな・漢字以外を含む場合はエラー

## 認識診断モード

公開URLの末尾に `?recognitionDebug=1` を付けると、設定画面に「認識診断」が表示されます。

端末内に最大80件まで、筆跡、各認識器の候補、得点、拒否理由、画数、空間分割、濁点判定、使用可能フォントなどを保存できます。診断JSONはエディターや外部サービスへ自動送信されません。
