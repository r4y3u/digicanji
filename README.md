# DigiCanji

漢字の読みを見て、手書きで適切な表記を答える学習アプリの試作版です。既存の手書き認識・字形判定処理と黒板調のUIを引き継ぎ、CSV問題、見直し、学習記録、縦書きレイアウト、リザルト画面を追加しています。

## 主な機能

- CSVから問題を生成
- 解答の文字数に合わせて認識欄を自動分割
- 問題・文字枠ごとの筆跡と認識文字を一時保存
- 「進む／回答」と「見直し」による問題移動・復元
- 認識欄クリックまたはキーボード矢印キーで文字枠移動（←→）、見直し（↑）、進む／回答（↓）
- 1回につき最大10問、重複なしで出題
- 基本はランダムとしつつ、正答率が低い問題と未出題問題を優先
- 累計出題回数と正解数をブラウザの `localStorage` に保存
- 横書き、縦書き・ワイド、縦書き・ポータブル
- 右利き／左利きに応じたパネル配置
- ポータブルでは横向き時に3分割レイアウトへ切り替わり、表示を90度回転し、入力座標も逆回転して認識へ渡す
- 1問ずつ判定を表示するリザルト演出
- 結果文のクリックによる解説オーバーレイ
- 設定画面からCSVファイル読込・CSV貼り付け・学習記録消去

## 問題CSV

UTF-8のCSVを使用します。列順は固定です。

```csv
前部,問部,後部,解答,解説
改善を,ハカル,。,図る。,しようと考えること
,アツイ,本を読む。,厚い,物を横から見たときの幅がある
選手の,コウタイ,を知らせる。,交代,人の役割などを入れ替えること
```

内部では各問題の識別子を内容から生成し、次の値を別途管理します。

- 累計出題回数
- 正解数

解答末尾の句読点と後部冒頭の句読点が同じ場合、重複を避けるため、その句読点は手書き対象から自動的に除外されます。上の1問目では、解答欄は「図る」の2文字になります。

CSVセル内にカンマ、改行、ダブルクォートを含める場合は、標準CSVと同様にセルをダブルクォートで囲んでください。

## 出題優先度

各問題に、正答率・出題回数・未出題かどうかを使った重みを付け、重み付きランダムで最大10問を重複なしに選びます。そのため、毎回固定順にはならず、正答率が低い問題ほど選ばれやすくなります。

## 起動方法

`file://` ではなく、簡易HTTPサーバー経由で開いてください。

```bash
python3 -m http.server 8080
```

ブラウザで次を開きます。

```text
http://localhost:8080/
```

GitHub Pagesでは、このフォルダーの内容を公開対象のルートへ置けます。

## 手書き認識

1. 対応ブラウザでは Handwriting Recognition API を試します。
2. 利用できない場合は Google Input Tools 形式へフォールバックします。
3. 画数、筆画のつながり、主要部品、候補の安定性を確認してから表示します。

フォールバック認識にはネットワーク接続が必要です。認識サービスの仕様変更やネットワーク制限の影響を避けたい本格運用では、固定の認識エンジンを別途組み込む構成が安全です。
